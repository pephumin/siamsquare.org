<h3>Installation</h3>

<p><a name="ipermissions"><b>What are the correct permission settings for phpESP files?</b></a></p>
<p>In a Unix environment, phpESP files should be set as 644 to be readable by Apache. For tighter security, <code>phpESP.ini.php</code> should be set 640, owner root and it's group set to the Apache process's group.</p>

<p><a name="iunsupported"><b>I'm getting an error that says PHP is running in a unsupported mode.</b></a></p>
<p>This error is an indication that your web server is running PHP in CGI mode instead of SAPI. phpESP can function in this configuration but authentication must be disabled. (<a href="#idisableauth">See disabling authentication in this FAQ.</a>) (<a href="http://us4.php.net/manual/en/install.general.php">See the PHP manual for more information on CGI VS SAPI.</a>)</p>

<p><a name="idisableauth"><b>Can I disable authentication for the management interface?</b></a></p>
<p>Authentication can be disabled by setting auth_design to false in phpESP.ini.php. Warning, making this change could pose a security risk.</p>
<p>Change this:</p>
<p>
<code>$ESPCONFIG['auth_response'] = true;</code><br>
<code>$ESPCONFIG['auth_design'] = true;</code>
</p>
<p>To this:</p>
<p>
<code>$ESPCONFIG['auth_response'] = false;</code><br>
<code>$ESPCONFIG['auth_design'] = false;</code>
</p>

<p><a name="iglobals"><b>Does register globals need to be on?</b></a></p>
<p>No.</p>

<p><a name="iblank"><b>I'm running Windows and keep getting a blank screen - Help!</b></a></p>
<p>If is is not a path problem in <code>phpESP.ini.php</code>, then copy <code>gnu_gettext.dll</code> (or <code>libintl-1.dll</code>) and <code>php_gettext.dll</code> (both usually found in the PHP install directory) into the system directory of windows. For Win95/98 the system directory is something like <code>c:\windows\system</code> and on WinNT/2000/XP the system directory is something like <code>c:\windows\system32</code>.</p>
<p>After you have copied these dll's to the proper place, edit your php.ini file and comment out the line "<code>extension=php_gettext.dll</code>". <br>
<br>Then stop and restart the web server.</p>

<p><a name="ipasswd"><b>Where do I set the MySQL login password?</b></a></p>
<p>The MySQL server name, user, and password that phpESP will use are set in the file phpESP.ini.php -- further details are in the INSTALL file.</p>

<p><a name="iotherdb"><b>Does phpESP support databases other than MySQL?</b></a></p>
<p>As of version 1.7 database abstraction has been implemented using the <a href="http://adodb.sourceforge.net/">ADOdb</a> library. MySQL is the most fully tested, Postgres has not been fully tested and SQLite has some known issues. We would welcome help making Postgres and SQLite support as strong as MySQL.</p>

<p><a name="ilanguage"><b>I'm having trouble changing the default language, what should I do?</b></a></p>
<p>Check the results of the phpESP system test page, then see the TRANSLATIONS file.</p>

<h3>Survey Management</h3>
<p><a name="mdelete"><b>Can I delete old surveys?</b></a></p>
<p>Yes, there is an unlisted function <code>admin/manage.php?where=purge</code>. Be <em>very</em> careful when using it, you cannot recover from it.</p>

<p><a name="monequestion"><b>How can I display only 1 question per page?</b></a></p>
<p>You can put a section break between each question to emulate this behavior.</p>

<p><a name="mrank"><b>Can I change the number of options on a rank question?</b></a></p>
<p>Yes, change the length field.</p>

<p><a name="mmultiple"><b>Can I prevent multiple votes (ballot stuffing)?</b></a></p>
<p>Well, if you give each respondent a username and password and have them login, you can limit multiple voting. If you don't force respondents to log in, it becomes more difficult (e.g., you could analyze the data by IP.)</p>

<p><a name="mlogin"><b>Can I limit the number of times someone takes a survey if they have to login?</b></a></p>
<p>Yes, set max login to the number of times you want to allow someone to take the survey. This may be done from the <em>Survey Access</em> page.</p>

<p><a name="mpart"><b>Can a respondent complete part of a survey, log out, then log back in later to finish?</b></a></p>
<p>Yes, see the <em>Save/Restore</em> option on the <em>Survey Access</em> page.</p>

<br><br>

<h3>Survey Deployment</h3>

<p><a name="dhandler"><b>What are <code>handler.php</code> and <code>phpESP.first.php</code>, and how do I use them?</b></a></p>
<p>They are used to embed a survey in another block of html/php code. You should always include phpESP.first.php at the top of your html code, before any output begins. Look at the examples in the example dir for usage.</p>

<p><a name="dresults"><b>How do I publish survey results?</b></a></p>
<p>You can make public results by doing the following, though note that it will make all results public. Make a copy of <code>public/survey.php</code> then edit your copy to uncomment approximately on line 48:</p>
<p><code>$results = 1;</code></p>
<p>then use your copied/edited page just like the survey.php page. It should then display results instead of the survey. The same applies to custom survey templates, setting $results to one before calling handler.php will switch to results mode.</p>

<h3>General Questions</h3>

<p><a name="glarge"><b>Can phpESP support large numbers of respondents?</b></a></p>
<p>Yes, large surveys (100 questions) and large respondent groups (4,000+) can be handled without a problem.  Throughput issues are generally a function of the speed of the web server, the speed of the MySQL server, and available network bandwidth.</p>
